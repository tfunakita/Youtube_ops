## 1. ファネル構造とワクワク設計

> 🚧 **作業中** — 概念レベルのたたき台。実例が溜まり次第、shoyatecと共同で更新する。

### 基本思想
> 「すべての行動は人の"ワクワク"が起点になっている」

視聴者がYouTubeを見てスクールを買うまでの全プロセスは、論理ではなく感情（ワクワク）が先に動き、あとから理屈がついてくる。

### ファネル × ワクワクの質

| ステップ | アクション | ワクワクの質 | 起点 |
|---------|-----------|------------|------|
| 入口 | サムネ・タイトルをクリック | 「なんか気になる」（0.1秒の引力） | CTR |
| 中間 | 動画を最後まで見る | 「自分ごとになってきた」（共感） | 視聴維持率 |
| 出口① | LINE登録する | 「もっと知りたい」（期待） | LINE登録数 |
| 出口② | スクール購入 | 「未来が見えた」（確信） | 成約数・売上 |

### ワクワク設計の3種類

**① 視聴者のワクワク**
サムネを見た瞬間「これ見たい」となる引力。CTR直結。数字・具体性・顔の表情が効く。

**② 作り手のワクワク**
MCやスタッフが「これ面白そう」と思えるかどうか。出演者の熱量は画面に出る。企画の段階でここを外さない。

**③ 担当者の嗅覚**
データで「正しい」だけでなく、「これ勝ちそう」という直感で選ぶ企画。これがAIに代替できない核心。

## 2. MTG診断フロー

ファネルのどこが詰まっているかを診断し、打ち手を決める。

### A. 入口が詰まっている（再生数 △・×）

**原因① CTRが低い（サムネ・タイトルの問題）**
- サムネに人の顔（感情・表情）を入れていない
- タイトルに数字・具体性がない
- 競合の高CTRサムネを分析して真似る
- A/Bテスト：同じ動画でサムネを途中差し替えて比較する

**原因② 投稿本数が足りない**
- 週1本を週2本に増やす
- ショート動画で露出を増やす（長尺への誘導ではなくチャンネル認知目的）
- 企画のストックを作る

**原因③ ジャンル・テーマがアルゴリズムに乗っていない**
- 伸びている競合チャンネルの直近ヒット動画のテーマを真似る
- 検索需要があるキーワードからテーマを逆算する
- シリーズもの・続きものにして関連動画に乗りやすくする

---

### B. 中間が詰まっている（再生数○ / 視聴維持率 △・×）

**原因① 冒頭で離脱している（最初の30秒問題）**
- 冒頭に「この動画で得られること」を明示する
- 結論を先に言ってから解説する構成に変える
- 冒頭の尺を短くする（挨拶・前置きを削る）

**原因② 動画の途中で離脱している**
- テロップ・BGM・カット編集でテンポを上げる
- 1動画のテーマを絞る（詰め込みすぎない）
- 視聴者が知りたいことだけに絞って不要な説明を削る

**原因③ 動画の内容と検索意図がズレている**
- タイトルで約束したことを動画内で答えているか確認する
- コメント欄で「思ってたのと違う」系の反応がないか確認する

---

### C. 出口が詰まっている（再生数○ 維持率○ / LINE △・×）

**原因① CTAが弱い・遅い**
- CTAを動画の中盤（50〜70%地点）にも入れる
- 「LINE登録してください」ではなく「〇〇が欲しい人はLINEへ」と具体的にする
- 概要欄のLINEリンクを上部に移動する

**原因② 視聴者とオファーがズレている**
- 動画の視聴者層（ペルソナ）とLINE登録後のオファーが合っているか確認する
- 認知動画ばかりで顕在層向け動画が少ない場合は比率を変える

**原因③ LPの問題（LINE登録後の離脱）**
- LINE登録ページ（LP）のファーストビューが弱い
- 登録後のメッセージがすぐに来ない・内容が薄い

---

### D. 売上○ だが再生数 △・×（将来リスク）

- 認知動画の投稿比率を上げる
- 既存のCV動画に頼りすぎているサイン → 新規流入を増やす施策を優先
- チャンネル登録者数の推移を確認（横ばい・減少なら要注意）

## 3. AI活用ガイド

> 🚧 **作業中** — 概念レベルのたたき台。実業務での検証を経て更新する。

### 基本方針
> 「情報処理・構造化・たたき台 → AI　/ 判断・関係・感覚 → 人間」

AIを使いこなす能力自体にも差が出る時代。「何をAIに任せるか」の判断力が新しい差になる。

### 業務別 AI活用度

| 業務 | AI活用度 | 備考 |
|------|---------|------|
| KPI集計・自動更新 | ✅ 完全自動 | 毎日10時 GitHub Actions |
| 月次レポート生成 | ✅ ほぼ自動 | データ→文章化 |
| 競合リサーチ・トレンド調査 | ✅ 高い | プロンプト次第で精度上がる |
| 企画案のたたき台 | ✅ 高い | ベースをAIで出して人間が磨く |
| 台本構成 | ✅ 高い | フォーマットが決まっていれば |
| PDCA・戦略立案 | 🟡 中 | データは読めるが勝ち筋の嗅覚は人間 |
| 企画の「ワクワク」判断 | ❌ 人間 | AIが出したものを人間が選ぶ |
| サムネ・タイトルの最終判断 | ❌ 人間 | チャンネルの文脈・視聴者感覚 |
| クライアント関係構築 | ❌ 人間 | 信頼・感情・空気感 |
| 撮影・出演 | ❌ 人間 | 人間そのもの |
| クライアントのモチベ管理 | ❌ 人間 | 人間関係 |

### AIに任せる際の注意
- AIの使い方（プロンプトの質・情報の構造化）自体も標準化する
- アウトプットは必ず人間が検証する
- 「AIが出したから正しい」は禁止

## 5. KPI管理

### ファネル指標

| 指標 | 意味 | 判定基準 |
|------|------|---------|
| 再生数/本 | 認知の広がり | ○達成 / △50〜70% / ×未満 |
| 投稿本数 | 露出量 | 目標本数に対する達成率 |
| LINE登録数 | 関心の深さ | ○100% / △50% / ×未満 |
| 成約数 | 購買 | ○100% / △70% / ×未満 |
| 売上 | 最終KPI | Q目標から逆算したストレッチ目標 |

### 今月目標のストレッチ計算
```
今月目標 = (Q目標 − 今月より前の実績合計) ÷ 残り月数（今月含む）
```
